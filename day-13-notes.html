01 · Overview

What is a nested if?

A nested if is simply an if statement placed inside another if (or elif, or else) block. It's how you check a second condition, but only after the first one has already turned out True.

Think of it as a follow-up question — you only ask it once the first question has already been answered "yes."

02 · Syntax

How to write it

The inner if is just an ordinary if statement — it just happens to live inside the indented block of an outer one. Each level of nesting adds another 4 spaces of indentation.

syntax.py
if outer_condition: # runs only if outer_condition is True if inner_condition: # runs only if BOTH conditions are True statement

Example — checking age and citizenship to vote

voting.py
age = 20 is_citizen = True if age >= 18: if is_citizen: print("You are eligible to vote.") else: print("You must be a citizen to vote.") else: print("You are too young to vote.")
▸You are eligible to vote.

03 · Nested if vs and

When to nest, and when not to

If you only care about both conditions being True together, a single if with and (from Day 9) is simpler than nesting. Nesting is more useful when each level needs its own separate else — like the voting example, where "too young" and "not a citizen" are two different messages.

Use and

if age >= 18 and is_citizen: — when there's just one combined outcome to check.

Use nesting

When each condition needs its own else branch with a different response.

04 · Multiple Levels

Nesting can go more than one level deep

There's no hard limit on how many levels you can nest — but each level adds another layer of indentation, and the code gets harder to follow fast. Two levels is usually fine; beyond that, it's often a sign to simplify with elif or functions.

login_system.py
username = "admin" password = "1234" is_active = True if username == "admin": if password == "1234": if is_active: print("Login successful.") else: print("Account is inactive.") else: print("Incorrect password.") else: print("Username not found.")
▸Login successful.

🧪 Try it yourself

Create variables for a number, then write a nested if that first checks if it's positive, and — only if it is — checks whether it's even or odd.
